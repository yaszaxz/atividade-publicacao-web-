Atividade final publicação web


Publicação web estática

É uma publicação web que não tem interação com o usuário, ela apenas exibe informações.
  • O conteúdo é fixo e não muda automaticamente
  • Normalmente é composta por páginas html simples com css
  • Menor interatividade
  • Fácil de publicar

Publicação web dinâmica

É uma publicação web que tem interação com o usuário em tempo real. Ela se altera com base na interação do usuário com a publicação.
  • O conteúdo pode ser alterado pelo usuário
  • Suporta banco de dados para armazenar informações
  • Desenvolvimento e manutenção mais complexas
  • Tem interatividade

Diferenças

  • Em uma página estática o conteúdo é fixo, já na dinâmica não
  • Páginas estáticas não usam banco de dados ou scripts, já as dinâmicas, sim
  • Baixa interatividade nas estáticas e alta nas dinâmicas
  • Manutenção mais simples nas estáticas e mais complexa nas dinâmicas

Domínio e DNS

Domínio

Um domínio é o nome que identifica de forma única um site ou serviço na internet. Ele permite com que usuários acessem recursos de forma mais simples, sem precisar digitar o endereço de ip da página.

DNS

A sigla DNS significa Domain Name System. O DNS serve para converter o link do site que está em um domínio para um endereço de ip numérico.

Servidores compartilhados e dedicados

Compartilhados

É um tipo de hospedagem onde vários sites ou clientes dividem recursos de um único servidor físico. Em um servidor compartilhado os sites hospedados compartilham a mesma infraestrutura, como a CPU, memória, espaço em disco e banda de rede.

Vantagens

  1. Custo mais baixo
  2. Facilidade de uso
  3. Suporte técnico
  4. Gestão simplificada

Desvantagens

  1. Desempenho varável
  2. Limitações de recursos
  3. Menos controle
  4. Segurança comprometida

Dedicados

É um tipo de hospedagem onde você aluga um servidor inteiro para uso próprio. Diferente do servidor compartilhado, você tem total conrole sobre recursos.

Vantagens

  1. Desempenho superior
  2. Maior controle e personalização
  3. Segurança aprimorada
  4. Escalabilidade

Desvantagens

  1. Custo elevado
  2. Necessidade de conhecimentos técnicos
  3. Manutenção e administração por conta própria

Servidores

O que é um servidor web?

Servidor web é um software ou hardware que armazena conteúdos da web para os usuários, incluindo páginas html, imagens e vídeos e outros recursos, sendo acessado pelos usuários através do navegador. O servidor web é responsável por processar as requisições de um usuário e enviar a resposta adequada, geralmente no formato de uma página web.

Apache HTTP server

O Apache é um dos servidores web mais populares e amplamente utilizados no mundo e é a escoha padrão para muitos sites e servidores.

Principais características

  • Configuração flexível
  • Suporte a módulos
  • Suporta várias linguagens de programação

Vantagens

  1. Grande comunidade e documentação
  2. Alta flexibilidade
  3. Compatibilidade

Desvantagens

  1. Desempenho
  2. Consumo de recursos

Ngix

É um servidor web que tem se destacado pelo seu desempenho e escalabilidade, principalmente em sites de alto tráfego. Ele começou como um servidor de proxy reverso, mas também é amplamente utilizado como servidor web.

Principais características

  • Desempenho e escalabilidade
  • Proxy reverso e balanceamento de carga
  • Baixo consumo de recursos

Vantagens

  1. Desempenho
  2. Função de proxy reverso e balanceamento de carga
  3. Alta escalabilidade

Desvantagens

  1. Menos flexível que o Apache
  2. Suporte a CGI limitado

Como fazer e restaurar backup

  1. Backup Compactado: A compactação de arquivos é uma excelente maneira de economizar espaço e garantir que seus dados estejam protegidos em caso de falhas. No Linux/macOS, usamos o comando tar com a opção -z para compactar com gzip. No Windows, o PowerShell com Compress-Archive é a ferramenta mais comum para compactação.
  2. Restaurar Backup: Para restaurar, no Linux/macOS, usamos o comando tar -x para extrair arquivos de um backup .tar.gz, e no Windows, usamos o PowerShell com o comando Expand-Archive.

Conhecimentos

  • Matérias escolares
  • Conhecimento mínimo de programação
  • Inglês intermediário para avançado

Servidor de banco de dados e backup

O que é um servidor de banco de dados

Um servidor de banco de dados é um sistema que armazena e gerencia bases de dados, permitindo que os usuários e aplicações acessem, modifiquem e manipulem dados de forma eficiente e segura.

Como fazer o backup em MySQL

O backup geralmente é feito usando a ferramenta mysqldump, que cria uma cópia do banco de dados em um arquivo SQL. O arquivo resultante pode ser restaurado posteriormente, se necessário.
  1. Comando para criar um backup: mysqldump -u usuario -p nome_do_banco > backup.sql

Como restaurar o backup

Para restaurar um banco de dados no MySQL a partir de um arquivo .sql gerado pelo mysqldump, você pode usar o comando mysql.
  1. Comando para criar um backup: mysql -u usuario -p nome_do_banco < backup.sql

Serviço SSH

O que é

SSH (Secure Shell) é um protocolo de rede criptografado que permite a comunicação segura entre dois computadores (geralmente, entre uma máquina local e um servidor remoto) através de uma rede não segura, como a internet. O serviço SSH permite a autenticação e a execução de comandos remotos em um servidor, além de fornecer um canal seguro para a transferência de arquivos. É possível também transferir arqvuivos de uma máquina para a outra

Como o SSH permite o acesso

O SSH proporciona acesso remoto seguro a um servidor web por meio de criptografia de dados e autenticação de usuário, garantindo que as informações trocadas entre o cliente e o servidor estejam protegidas. Ele permite a execução de comandos remotos e a transferência de arquivos com segurança, sendo amplamente utilizado para administração e manutenção de servidores web em ambientes de produção.

Linux

O que é Linux

Linux é um sistma operacional de código aberto e gratuito para instalação e uso. Por ser de código aberto, várias distribuições e versões foram criadas pela comunidade para atender diferentes necessidades.

Principais distribuições

  • Ubuntu Server
  • CentOS
  • Debian
  • Fedora Server
  • Arch Linux
  • openSUSE

Comandos básicos do Linux

  • pwd: Mostra o caminho completo do diretório atual
  • ls: Lista o conteúdo de um diretório
  • cd: Muda o diretório citado
  • mkdir Cria um diretório no diretório atual
  • cp Copia arqvuivos ou diretórios
